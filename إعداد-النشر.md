# إعداد النشر — HTTPS، الدومين، و SSH

## ١) تثبيت HTTPS نهائياً

### إذا الموقع على GitHub Pages
- **HTTPS مفعّل تلقائياً** ولا تحتاج تثبيت شهادة.
- استخدم دائماً الرابط: `https://has1992.github.io/sinad/`

### إذا استخدمت دومين خاص (مثل sinadcity.com)
1. ادخل المستودع: **https://github.com/Has1992/sinad**
2. **Settings** → **Pages**
3. في **Custom domain** اكتب: **sinadcity.com** → **Save**
4. فعّل خيار **Enforce HTTPS** ✓ (يظهر بعد أن يصبح الدومين مفعّلاً)
5. في لوحة تحكم الدومين اضبط السجلات كما في القسم التالي

## ٢) ربط www مع الدومين (مثال: sinadcity.com)

لأن يعمل كل من **sinadcity.com** و **www.sinadcity.com**:

### في GitHub (Settings → Pages → Custom domain)
- اكتب: **sinadcity.com** (أو **www.sinadcity.com** إذا أردت أن يكون المدخل الرئيسي عبر www)

### في مزود الدومين (DNS) لـ sinadcity.com
اضبط السجلات كالتالي:

| النوع | الاسم (Name/Host) | القيمة (Value/Points to) |
|--------|-------------------|---------------------------|
| **A**  | `@`               | `185.199.108.153`         |
| **A**  | `@`               | `185.199.109.153`         |
| **A**  | `@`               | `185.199.110.153`         |
| **A**  | `@`               | `185.199.111.153`         |
| **CNAME** | `www`          | `Has1992.github.io`       |

- **@** = الدومين الرئيسي (sinadcity.com)
- **www** = النطاق الفرعي (www.sinadcity.com)

بعد الحفظ انتظر 5–30 دقيقة ثم جرّب **https://sinadcity.com** و **https://www.sinadcity.com**.

## ٣) إصلاح SSH و Permission denied

حتى يعمل `git push` بدون كلمة مرور (باستخدام المفتاح):

### 1) عرض المفتاح العام على جهازك
في الطرفية:
```bash
cat ~/.ssh/sinad_key.pub
```
انسخ السطر كاملاً (يبدأ بـ `ssh-rsa`).

### 2) إضافة المفتاح إلى GitHub
1. ادخل: **https://github.com/settings/ssh/new**
2. **Title:** اسم الجهاز (مثل Mac أو جهاز العمل)
3. **Key type:** Authentication Key
4. **Key:** الصق المفتاح الذي نسخته
5. **Add SSH key**

### 3) التأكد من إعداد SSH للمشروع
تأكد أن الملف `~/.ssh/config` يحتوي على:
```
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/sinad_key
  IdentitiesOnly yes
```

### 4) اختبار الاتصال
```bash
ssh -T git@github.com
```
المتوقّع: `Hi Has1992! You've successfully authenticated...`

### 5) ربط المشروع بـ GitHub عبر SSH
```bash
cd "/Users/one/Desktop/رمضان"
git remote set-url origin git@github.com:Has1992/sinad.git
git push
```

إذا ظهر **Permission denied** مرة أخرى:
- تأكد أنك أضفت المفتاح لحساب **Has1992**
- تأكد أن المستودع **sinad** موجود تحت هذا الحساب
- جرّب: `ssh-add ~/.ssh/sinad_key` ثم `ssh -T git@github.com`
